# v0.04-13 Usability Capstone

Part of **v0.04-13** (deferred until after [v0.04-14](../v0.04/v0.04-14-design-system-ux-migration/spec.md)). Presentation-layer additions that make the manual store-stock path usable without internal IDs:

```text
Demand → sourcing → draft PO → submit → vendor shipment → match → post → customer-ready
```

No schema or domain rule changes — composes existing services only.

## Screens

| # | Screen | Route |
|---|--------|-------|
| 1 | Buyer Action Queue | `/orders/buyer_workbench` |
| 2 | Demand Detail Supply + Next Action | `/demand/demand_lines/:id` |
| 3 | Sourcing Vendor Cards / Response Wizard Lite | `/sourcing/runs/:id` |
| 4 | Demand-to-PO Builder | `/orders/purchase_orders/from_demand` |
| 5 | PO Planned Coverage + Submit Impact | `/orders/purchase_orders/:id` |
| 6 | Vendor Shipment Receiving Entry | `/orders/receipts/new?receiving_mode=vendor_shipment` |
| 7 | Receipt Matching Workpad | `/orders/receipts/:id/edit` |
| 8 | Receipt Post Customer-Ready Checklist | `/orders/receipts/:id` (posted) |
| 9 | Item Drawer Demand-Native Lists | Items variant ops drawer |

## Acceptance summary

* Buyer workbench tabs with counts; row contract (demand, customer/purpose, supply one-liner, next action, primary CTA); bulk → PO builder
* Demand show: supply metric strip, supply sections, button CTAs in next-action panel
* Sourcing: vendor capability cards; response wizard with labeled PO line selectors
* PO builder: multi-demand vendor grouping; draft PO with planned coverage
* Draft PO: planned demand header metric; submit impact panel
* Vendor shipment entry form; receipt edit workpad with manual confirm, unmatched qty, pre-post impact
* Posted receipt: customer-ready checklist hierarchy
* Variant drawer: availability header, unified open demand list, inbound PO context
