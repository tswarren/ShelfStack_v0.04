## Deferred (non-blocking)

- Redirect route helper names (`customers_customer_requests`, `orders_purchase_requests`) remain as compatibility aliases; v00410 verifier does not assert route-name absence. Optional cleanup in v0.04-11.
- Duplicate demand-allocation pickup claim is app-level only (no DB constraint); acceptable pre-production; consider row lock before multi-register production.
- `DemandLines::SurfaceNotifyLines` recalculates allocation status for matching `notify` demand lines when stock increases via adjustment; it does **not** auto-allocate or send notifications. Treat as queue/status refresh only until a dedicated notify-ready workflow is scoped.
- Staff contact events use `customers.update` (not legacy `customer_requests.contact`); `CustomerContactEvent` no longer references dropped request models.
