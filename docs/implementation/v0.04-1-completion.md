**Status:** In review

## Purpose

Collapse catalog metadata onto `products` so active create/read/update flows no longer require `CatalogItem`. Path B retains `catalog_items` and `catalog_item_identifiers` as quarantined legacy tables until v0.04-2+.

## Path B confirmation

- `catalog_items` and `catalog_item_identifiers` **retained** in schema.
- **No new** `CatalogItem` / `catalog_item_identifiers` rows in Add Item, external ProductBuilder, or buyback intake flows.
- **Exception:** Ingram vendor feed import may still create/update legacy catalog rows until v0.04-2.
- `products.catalog_item_id` remains optional deprecated bridge; legacy reads via `Items::LegacyProductIdentifierBridge`.
- Transitional scan lookup: `products.sku` + legacy identifier join (`Pos::LineLookup`, `ExternalCatalog::LocalIsbnMatch`).
- Catalog item admin routes (`CatalogItemsController` identifier CRUD, external metadata viewer) remain for legacy identifier maintenance until v0.04-2.

## Remaining `CatalogItem` runtime usage (allowed transitional)

Roughly 35 `app/` references remain, classified as:

- **Legacy identifier admin:** `CatalogItemsController`, setup modals, identifier forms
- **Legacy lookup bridges:** `LegacyProductIdentifierBridge`, `Pos::LineLookup`, `LocalIsbnMatch`
- **Ingram import:** still upserts catalog rows then copies to product (metadata fusion on product)
- **Deprecated builders:** `CatalogItemBuilder` / `StagedCatalogItemBuilder` if not fully removed — prefer `ProductBuilder`

Do not add new active metadata reads from `CatalogItem` in Items workspace presenters or index.
