## Deliverables

### Vendor capabilities (slice A)

* `vendors` capability columns with controlled enums and JSONB fulfillment methods
* `Vendors::CapabilityResolver`
* Setup vendor form + `vendor.capability_updated` audit
* Ingram wholesaler profile in seeds

### Sourcing next actions (slice C)

* Capability snapshot columns on `sourcing_attempts` (set at submit)
* `Sourcing::NextActionPresenter` on sourcing run detail

### Planned demand coverage (slice D)

* `purchase_order_line_demand_plans` table and model
* `Purchasing::CreateDemandCoveragePlans`, `UpdateDemandCoveragePlans`, `ReleaseDemandCoveragePlan`, `PurchaseOrderLineDemandPlanSummary`
* Wired into `BuildPurchaseOrderFromDemand` and `AddDemandToPurchaseOrder`
* Draft plans do not affect inbound availability

### PO destination gates (slice D2)

* `purchase_orders.order_purpose`, `ship_to_type`, nullable `ship_to_snapshot`
* `vendor_direct_fulfillment` in `DemandAllocation::ALLOCATION_KINDS` (enum only — no MVP row creation)
* `Purchasing::CustomerDirectPurchaseOrderGate` blocks store receiving/posting for customer-direct POs

### Inbound conversion (slice E)

* `Purchasing::ConvertDemandCoveragePlansToInbound` on PO submit
* Removed ad hoc inbound allocation from `BuildPurchaseOrderFromDemand`
* `DemandLines::SupplySummary` shows **Planned on order** from draft plans

### Receiving entry (slice F)

* Receipt origin/receiving_mode fields with backfill defaults
* `Receiving::CreateVendorShipmentReceipt` for vendor-shipment entry without header PO

### Receipt matching (slice G)

* `receipt_line_matches` table
* `Receiving::PoLineMatchCandidates`, `SuggestReceiptLineMatches`, `ApplyReceiptLineMatches`
* `Receiving::ReceiptPostingMatchAdapter` — normalized PO-backed view for posting
* `PostReceipt`, `UpdatePoLineQuantities`, `ConvertInboundFromReceipt` consume adapter output

### Demand impact preview (slice H)

* `Receiving::ReceiptDemandImpactPreview` integrated into `Orders::ReceiptShowPresenter`

### Verifier and golden path (slice I)

* `lib/shelfstack/v00413_verify.rb` + rake task
