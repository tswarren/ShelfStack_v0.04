# v0.04-5 Used Variant Rules — Completion Note

## Status

**Complete** — 2026-06-30. Delivered on `v0.04-5-used-variant-rules` (PR #10 to `main`).

## Summary

Centralized new vs used variant behavior in `ProductVariants::OperationalPolicy` and enforced it across purchasing/TBO, buyback variant creation, customer-request guardrails, and item workspace UI—without demand tables, schema migrations, or POS lookup changes.

---

## Deliverables

### Policy layer

- [`ProductVariants::OperationalPolicy`](app/services/product_variants/operational_policy.rb) — context-aware booleans, block reasons, vendor sourcing applicability, operational badges
- Explicit distinction: `variant.orderable` (staff flag) vs `policy.vendor_orderable?` (computed operational eligibility)
- v0.04-5 default: non-new conditions treated as used-like for vendor-ordering rules

### Purchasing and TBO

- [`Purchasing::OrderEligibilityResolver`](app/services/purchasing/order_eligibility_resolver.rb) delegates to policy; blocks used-like at `:tbo` (behavior change from Phase 8.5-3)
- [`PurchaseRequests::CreateSingleLine`](app/services/purchase_requests/create_single_line.rb) blocks used-like TBO create
- TBO queue rows remain visible with disabled checkbox and Blocked badge ([`_tbo_row.html.erb`](app/views/orders/purchase_orders/_tbo_row.html.erb))

### Buyback

- [`Buybacks::FindOrCreateGradedUsedVariant`](app/services/buybacks/find_or_create_graded_used_variant.rb) uses `ProductVariants::SkuAllocator` (211 segment) instead of `SkuGenerator.variant_sku`

### Customer requests

- [`CustomerRequests::StartFromItem`](app/services/customer_requests/start_from_item.rb) blocks `special_order` for used-like variants
- [`ItemOperationsPresenter`](app/presenters/items/item_operations_presenter.rb) omits special order action and TBO/Order actions when not vendor-orderable

### Item UI

- Used-like variants emit info warning only (no missing vendor/source/cost warnings) via [`Items::OperationalWarningBuilder`](app/services/items/operational_warning_builder.rb)
- Operational badges on overview, operations table, and variant ops drawer
- Variant setup: disabled preferred vendor with explanation for used-like; orderable warning when checked

### Verification

- [`lib/shelfstack/v0045_verify.rb`](lib/shelfstack/v0045_verify.rb)
- [`shelfstack:v0045_verify_used_variant_rules`](lib/tasks/shelfstack/v0045_verify_used_variant_rules.rake) — report-only by default; `STRICT=1` merge gate
- Optional dev repair: `shelfstack:v0045_repair_orderable_flags`
