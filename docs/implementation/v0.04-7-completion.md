# v0.04-7 Allocations and Reservations — Completion Note

## Status

**In review** — 2026-07-01. Branch `v0.04-7-allocations-and-reservations` (not merged to `main`).

## Summary

Introduces **`demand_allocations`** as the v0.04 supply-claim model for on-hand and inbound PO quantity. Active on-hand allocations update **`inventory_balances.quantity_reserved`** / **`quantity_available`** via scoped **`Inventory::RebuildAvailabilityCache`** — never **`Inventory::Post`**. Hold-from-item auto-allocates partial/full stock; zero available skips allocation. Legacy `inventory_reservations`, `purchase_order_line_allocations`, and `receipt_line_allocations` are not written from v0.04-7 paths.

## Review fixes (2026-07-01)

Post-review hardening before merge:

- **Demand quantity guard** — `AllocateOnHand` and `AllocateInboundPurchaseOrder` reject quantity above `unallocated_quantity`; override applies to stock only.
- **Hold race** — `StartFromItem` returns `allocation_result: :none` on insufficient stock under lock.
- **Lock order** — allocation mutations lock `demand_line` then `demand_allocation` via `MutationSupport.lock_demand_and_allocation!`.
- **Inbound eligibility** — requires receivable PO header status (`submitted` / `partially_received`).
- **Used-wanted inbound** — service-level rejection.
- **Verifier** — expanded service scan, legacy + v0.04 inbound claim check, system user presence check.
