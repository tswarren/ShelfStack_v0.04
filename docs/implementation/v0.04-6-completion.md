# v0.04-6 Demand Foundation — Completion Note

## Status

**Complete** — 2026-07-01. Delivered on `v0.04-6-demand-foundation` (PR #11 to `main`).

## Summary

Introduces **`demand_lines`** and **`stock_considerations`** as the v0.04 runtime path for staff-entered demand. New demand creation from the item drawer, customer workspace, and `/demand` workspace writes **`demand_lines` only** — no dual-write to legacy `customer_requests`, `special_orders`, or `purchase_request_lines`. Hold actions record **intent only** (no inventory reservation until v0.04-7).

## Review focus

Reviewers should confirm:

1. **Write-path cutover** — drawer, customer workspace, and `/demand` create paths call v0.04 services only; no legacy row creation.
2. **Presentation cutover** — drawer does not offer legacy TBO/Order creation alongside v0.04 demand actions; legacy activity is collapsed, not primary.
3. **Eligibility matrix** — used wanted, special order, manual TBO, buyer replenishment blocked per spec; delegates to v0.04-5 policy.
4. **No inventory side effects** — create, match, cancel, expire, stock consideration convert/dismiss do not call `Inventory::Post`.
5. **Verifier** — `STRICT=1` enforces static and data checks documented in spec.

Legacy routes (`/customers/customer_requests`, legacy TBO-to-PO screens, item operations **Mark TBO / Add to PO** header actions) remain reachable elsewhere — removal deferred to v0.04-10.

## Manual smoke (reviewers)

1. **Item drawer** — open variant ops → **Record hold request** → confirm `demand_line` on `/demand`; no legacy customer request; no reservation.
2. **Used wanted** — used-like variant + customer → demand line with `used_wanted` intent; no PO/TBO linkage.
3. **Manual TBO** — vendor-orderable variant → demand line on `/demand`; not on legacy build-from-TBO screen.
4. **Stock consideration** — create → convert → linked demand line; dismiss path for terminal statuses.
5. **Customer show** — **New demand** pre-fills customer; demand lines table populated.
6. **Legacy activity** — existing v0.03 requests/holds visible only under collapsed **Legacy activity** in drawer.
