## Review fixes (post-implementation)

Addressed merge blockers from code review:

1. **Final-response gate** — inbound and vendor-backorder allocations apply only when `final_response == true`.
2. **Buyer review** — `quantity_failed` and unaccepted backorder quantity set `buyer_review_required`.
3. **Inbound linking** — confirmed PO responses link the allocation returned by `AllocateInboundPurchaseOrder`, not latest inbound on demand line.
4. **PO line validation** — `Sourcing::ValidatePoLineLink` on attempt create and confirmed allocation (same store, variant, receivable PO, eligible line, quantity when allocating).
5. **Eligibility** — sourcing uses `vendor_orderable?` (includes active product/variant).
6. **Close run** — cancels pending/submitted attempts before resolving run.
7. **Audit UI** — run show uses prepared `@audit_events` (run, attempts, responses).
8. **Docs** — data-model and test-plan status updated to **In review**.
