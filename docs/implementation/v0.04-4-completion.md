## Deliverables

### Routing and URL contract

- Canonical item URLs use `product_id` / `product_variant_id`; legacy `catalog_item_id` redirect shim preserved in `ItemsController#show`
- Item setup return paths, external metadata back links, and catalog identifier cancel links prefer `product_id` when linked
- [phase-9-item-drill-down-contract.md](../handoff/phase-9-item-drill-down-contract.md) updated

### Item workspace language

- Product-owned item setup copy (`Item metadata`, bibliographic details, product-first lifecycle `needs_product`)
- Removed dead `catalog_item_identifier` search hit branch
- Catalog-only shell messaging and integration test coverage

### Product-first intake hygiene (review fixes)

Discovered during manual smoke after core wire-through; closes gaps between transitional SKU cache and v0.04-2 identifier ownership:

- **`Items::PersistInitialProductIdentifier`** — creates primary `product_identifiers` row from intake ISBN/GTIN (or syncs from `products.sku` when appropriate)
- **`Items::BootstrapCatalogLinkedProduct`** — standalone catalog-item create with initial identifier bootstraps linked product + identifier in one transaction
- **Add Item wizard** (`save_product!`) — persists identifier after transitional SKU assign
- **`ProductsController#edit_metadata` / `#update_metadata`** — bibliographic edit for fused-metadata products without a linked `catalog_item`; legacy catalog-linked products redirect to catalog-item edit
- Item setup **Edit bibliographic details** link uses `ItemPresenter#edit_bibliographic_metadata_path` for both paths

### Buyback cleanup

- New intake/resolution paths stop writing `catalog_item_id` / `created_catalog_item_id`
- Removed dead `create_product_for_legacy_catalog!`
- Product-centric `CreateIntakeItem` / `ResolveItem` results and resolve UI copy

### External catalog / Ingram hygiene

- `ImportCandidate` uses internal `product_id`; deprecated `catalog_item_id` accepted at boundary only
- Preview duplicate links and hidden fields use `product_id`
- Staff-facing import label for `create_catalog_item` action: “Review and edit item details”

### Product SKU / defaults

- `SkuGenerator.product_sku` reads `product.primary_identifier` / `products.sku`
- `Product#apply_product_defaults` uses product identifiers, not catalog bridge

### Purchasing

- TBO format filter on `products.format_id` (no `catalog_items` join)
- `discontinued_product?` on fused product `publication_status`; `discontinued_product*` warning keys authoritative; `discontinued_catalog_item*` retained as aliases

### Customer request bridges

- `StartFromItem` / `MatchVariant` stop writing `catalog_item_id` on new lines (no demand behavior changes)

### Verification

- `lib/tasks/shelfstack/v0044_verify_wire_through.rake` — report-only by default; `STRICT=1` merge gate
