# v0.04-2 Product Identifiers — Completion Note

## Status

**In review** — submitted for final review (2026-06-29). Merge-gate fixes applied (ISBN `normalize_preview`, variant SKU fallback removal, house `201` enforcement, invalid alternate guard, Ingram product-first resolution, duplicate audit cleanup, internal EAN seed, house check-digit validation).

## Deliverables

### Schema and models

- `product_identifiers` with validation families (`gtin`, `isbn`, `freeform`, `house`) and partial unique indexes
- `internal_ean_sequences` with seeded `201`/`product_house` and `211`/`variant_sku` rows
- `product_variant_lookup_codes` for optional short POS lookup codes
- Models: `ProductIdentifier`, `InternalEanSequence`, `ProductVariantLookupCode`
- `Product#primary_identifier`, `#active_identifiers`; `ProductVariant` lookup code association

### Core services

- `ProductIdentifierService` — CRUD, ISBN alternates (978/979 rules), house generation, `products.sku` cache sync, audit events
- `InternalEanAllocator` — row-locked EAN-13 for segments `201` and `211`
- `ProductVariants::SkuAllocator` — new variant SKUs from segment `211` (create-time only; historical SKUs preserved)
- `ProductVariants::LookupCodeService` — normalize, CRUD, POS resolve
- `Items::ProductIdentifierLookup` — product-grain lookup (no legacy bridge)

### Backfill and verification

- `V0042::BackfillProductIdentifiers` migration from legacy catalog identifiers and bare `products.sku`
- Conflict policy: no silent skip; `needs_review` flag; stdout logging
- `rails shelfstack:v0042:verify_product_identifiers` (includes zero legacy app-reference gate)

### Scan resolution (mandatory)

- `Pos::LineLookup` — variant SKU → lookup codes → `product_identifiers` → legacy `products.sku` (no identifiers only)
- `Inventory::VariantLookup` — same order
- `Purchasing::LineLookup` — product identifier path for PO line matching

### Import and Ingram

- `ExternalCatalog::ProductBuilder` writes `product_identifiers` on create
- `AddItem::TransitionalSkuAssigner` conflict checks via `ProductIdentifierLookup`
- `IngramCatalogImport::Runner` writes identifiers on product upsert; no new catalog identifier rows

### UI (minimum)

- Setup modals: product-scoped identifier quick add/edit; generate house EAN (`201`)
- Item setup: product identifier table with barcode-safe styling for `gtin`/`house`
- Selling setup: variant SKU shown as barcode-safe value; lookup codes column (read-only)
- Identifier preview API uses `ProductIdentifierService`
- Catalog-item identifier routes delegate to product-scoped `ProductIdentifierService`
- Product SKU edit syncs primary `product_identifier` via `ProductIdentifierService.sync_from_product_sku!`
- Item setup identifier modal renders for product-first items (not only catalog-linked)
- Catalog item show: edit/remove identifier actions; house EAN gated on linked product
- Item setup identifier table: remove action via setup modals (when multiple identifiers)

### Legacy removal

- Dropped `catalog_item_identifiers` table
- Removed `CatalogItemIdentifier`, `CatalogIdentifierService`, `Items::LegacyProductIdentifierBridge`
- `CatalogItem` quarantined as metadata shell; identifiers owned by products only

### Tests and helpers

- Service/model tests for allocators and `ProductIdentifierService`
- `Phase3TestHelper` attaches test identifiers; `add_test_product_identifier!` for legacy catalog paths
- `V0042TestHelper` seeds internal EAN sequences in test setup
