# v0.04-6 Demand Foundation — Implementation Plan

**Status:** Ready for implementation / branch start

## Goal

**Begin runtime cutover** to `demand_lines` and `stock_considerations`. All new staff-facing demand creation writes **`demand_lines` only**. Item drawer uses **v0.04 demand actions only**.

Legacy tables/routes may remain physically present until v0.04-10; **do not expand legacy write paths**.

**Authoritative docs:** [spec.md](../docs/v0.04/v0.04-6-demand-foundation/spec.md) · [data-model.md](../docs/v0.04/v0.04-6-demand-foundation/data-model.md) · [test-plan.md](../docs/v0.04/v0.04-6-demand-foundation/test-plan.md)

## Decisions (locked)

| Decision | Choice |
| -------- | ------ |
| Legacy posture | Cutover start — new writes to `demand_lines` only |
| Item drawer | v0.04 demand actions **only**; no side-by-side legacy |
| Demand number | `{store_number}-D{sequence:06d}` via `DemandLines::NumberAllocator` |
| Expiry | `DemandLines::Expire` manual/staff in v0.04-6; nightly job → v0.04-7 |
| manual_tbo | `/demand` only; not legacy TBO-to-PO screens |
| used_wanted | Customer or snapshot required |
| Quantity fields | `quantity_requested` only; allocation qty fields → v0.04-7 |

## Hard gates

| In scope | Out of scope |
| -------- | ------------ |
| `demand_lines`, `stock_considerations` | `demand_allocations` (v0.04-7) |

| `/demand` primary UI | PO build from demand |
| Eligibility matrix | Sourcing (v0.04-8) |
| Manual expire | Nightly expire job (v0.04-7) |
| Cutover verify rake | `Inventory::Post` from demand |
